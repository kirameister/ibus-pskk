# 背景

## PSKKとは？

**PSKK**は**Personaliz(ed|able) SKK**の略であり、高度なカスタマイズ性と効率的なタイピング体験を求めるユーザー向けに設計された日本語入力方式エンジン（IME）です。クラシックなSKK（Simple Kana to Kanji conversion）に触発されていますが、漢直（直接漢字入力）、同時キー処理、CRFベースの文節分割などの最新技術と機能を統合することで、その基盤を拡張しています。

## 設計哲学

PSKKの開発は、3つのコア原則によって駆動されています。

1.  **完璧性よりも制御**: PSKKは文単位での完璧な変換を目指していません。代わりに、エンドユーザーにIME変換の完全な制御権を与えることを目指しています。
2.  **個人化**: 入力体験のあらゆる側面が設定可能であるべきです。キーバインディングとキーボードレイアウトから変換ロジック自体まで、PSKKは「あなた自身の」個人的なIMEであることを目指しています。
3.  **効率性**: 同時キープレスと直接漢字入力をサポートすることにより、PSKKは日本語テキストに必要なキーストローク数を削減し、より高速で流動的なタイピングを実現します。
4.  **実装の単純性、機能の力**: PSKKはPythonで実装されており、開発と拡張性を容易にしながら、文節境界予測などの高度な機能には`crfsuite`などの堅牢なライブラリを利用しています。

## コア機能とテクノロジー

### 1. CRFを用いた辞書駆動型変換
従来のIMEが複雑なヒューリスティック基盤の形態素解析に依存するのとは異なり、PSKKは辞書駆動型のアプローチを使用しています。複数の文節入力を効果的に処理するために、PSKKは**条件付き確率場（CRF）**を選択的に採用して、文節（句単位）の境界を予測し、より長い文の変換精度を向上させます。

### 2. 漢直（漢直 - 直接漢字入力）
PSKKは直接漢字入力をサポートしており、ユーザーは事前に定義されたキーの組み合わせ（通常、マーカーキーと一緒に保持される2つのキー）を通じて特定の漢字を出力できます。これにより、頻繁に使用される単語の通常のひらがな漢字変換プロセスをバイパスします。

### 3. 同時キー入力
エンジンは同時キープレス（日本語では「同時打鍵」と呼ばれることが多い）をサポートしており、これは**新下駄配列（Shin-Geta）**などの専門的な日本語キーボードレイアウトの重要なコンポーネントです。これにより、ひらがなのより人間工学的で高速な入力が可能になります。

### 4. SandS（Space and Shift）
パワーユーザーの間で人気のある機能で、SandSはスペースキーが押し続けると機能Shiftキーとして機能し、タップするとスペースキーとして機能することを可能にします。これにより、指の動きを最小化し、タイピング速度を向上させます。

### 5. ユニークな入力ステート
PSKKは、次を含む洗練されたステートマシンを通じて入力を管理します。
- **ステルスモード（IDLE）**: 入力がすでに確定しているかのように表示され、視覚的な気晴らしを軽減します。
- **文節モード**: テキストを変換するための明示的なマーク。
- **強制プリエディット**: 漢直で生成された漢字と後続の辞書変換用のひらがなを混合することを可能にするユニークなモードで、柔軟な入力戦略を可能にします。

## プロジェクト履歴

PSKKは「メンテナンス不要」のIMEを作成する試みとしてスタートしました。これは、日本語IMEが日々日本語を入力したいと考えている人々が使用しているにもかかわらず、過小評価され、過小投資されているという懸念に由来しています。

既存の日本語IMEのほとんどは、ひらがな漢字変換にために統計モデルに依存しています。自然言語は生きている存在であるため、そのような統計モデルには継続的なメンテナンスが必要です。このプロジェクトは、IMEのメンテナンス努力と制御の両方をエンドユーザーにシフトさせることで、この特定の問題に対処するために開始されました。

この目標を達成するために、SKKが開始点として選択されました。なぜなら、それは統計モデルに依存していないためです。既存の統計モデルを備えた日本語IMEと競争するために、同時タイピングとSandSと漢直が実装されました。その後、オプションの統計モデルが導入されましたが、それは正確な文節単位のひらがな漢字変換が辞書に見つからなかった場合のサポートメカニズム以外の何ものでもないことを意図していました。
